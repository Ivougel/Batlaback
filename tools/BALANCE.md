# Баланс игры

Актуально: 2025-06-23. Константы — `battle-engine.js`. Симуляция: `node tools/balance-sim.js` → `tools/balance-sim-results.json`.

## Глобальные ограничения (anti-scaling)

| Константа | Значение | Назначение |
|---|---|---|
| `POISON_STACK_CAP` | 3 | Максимум стаков яда |
| `POISON_DOT_PER_STACK` | 0.5 | DoT: floor(стаки × 0.5) HP/с (мин. 1) |
| `MAX_POISON_BONUS_PER_ITEM` | 2 | Бонус яда от синергий на предмет |
| `MAX_CDR_RATIO` | 0.5 | Мин. кулдаун 50% от базы |
| `MAX_DAMAGE_MULT_BONUS` | +58% | Потолок физ. множителя |
| `MAX_MAGIC_DAMAGE_MULT_BONUS` | +78% | Потолок маг. множителя |
| `DAMAGE_PACING_SCALE` | 0.88 | Сглаживание исходящего урона предметов |
| `POISON_STACK_PACING` | 0.55 | Сглаживание стаков яда при наложении (×1 может не пройти) |
| `GROUND_FIRE_PACING` | 0.85 | Сглаживание огня на поле |
| `STAMINA_BASE_MAX` | 40 | Базовый пул ⚡ |
| `STAMINA_REGEN_PER_SEC` | 5 | Базовая регенерация ⚡ (+1/с за каждое оружие с ненулевой ⚡) |
| `STAMINA_COST_SCALE` | 1.05 | Множитель стоимости активации (+ CD-floor под regen 6) |
| `MAX_LIFESTEAL` | 35% | Вампиризм |
| `MAX_CRIT_CHANCE` | 35% | Крит |
| `MAX_SHIELD_BLOCK_MULT` | 45% | Усиление блока |
| `MAX_FLAT_DAMAGE_BONUS` | 6 | Плоский урон от синергий |
| `MAX_FLAT_BLOCK_BONUS` | 10 | Плоский блок от синергий |
| `MAX_FLAT_HEAL_BONUS` | 6 | Плоское лечение |
| `ARMOR_ABSORB_CAP_RATIO` | 50% | Броня не поглощает >50% удара |
| `REPEAT_MAGIC_EFFECT_SCALE` | 50% | Повтор только у `mana_orb`, на себя |

## Элементальный пробой (magic / fire)

| | Блок | Броня | Cap брони за удар |
|---|---|---|---|
| **magic** | −40% эфф. блока | −55% DEF | 30% (вместо 50%) |
| **fire** | −45% | −60% | 30% |

DoT (яд, огонь на поле) бьёт напрямую по HP.

## Stacking блока

Несколько предметов с активным `block` на одной стороне:

1. Сортировка по силе блока (база + синергии).
2. Эффективность: **100% → 85% → 72% → 60%** (`BLOCK_SOURCE_EFFICIENCY`).

Самый сильный щит всегда на 100%. В логе: `+4 блок (85% от стака)`.

## Stacking яда

Несколько предметов с `poison` на одной стороне:

1. Сортировка по силе яда (база + poisonBonus).
2. Эффективность: **100% → 65% → 50% → 40%** (`POISON_SOURCE_EFFICIENCY`).

Cap стаков **×3**. DoT при ×3: **1 HP/с**. Одиночный +1 стак с pacing 0.55 часто не проходит (floor(0.55)=0).

## Лечение под ядом

Каждый стак яда на цели снижает лечение на **5%** (макс. **−50%** при ×10).

Яблоко (+3) под ×5 яда → **+2 HP** (штраф лечения). **Вампиризм** использует те же штрафы.

## Усталость арены (fatigue)

Порог зависит от **номера раунда** забега (1…16):

| Раунд | Старт усталости |
|---|---|
| 1 | ~18 с |
| 8 | ~12 с |
| 16 | ~7 с |

После порога (раньше на поздних раундах и при 6+ предметах на доске):

1. **+12% входящего урона**, +5% каждые 5 с (cap **+40%**).
2. Через **10 с** после старта — **−2 HP/с** обоим (анти-stall).

В prep-фазе показывается подсказка для текущего раунда и типичного числа предметов.

## Темп боя

- Стартовый кулдаун предметов: **50–100%** базы (разнос первого залпа).
- Не более **3 активаций** за tick на сторону (волны вместо one-frame burst).

## Footprint предметов

Сильные gems, руны и class legendaries (кроме уже крупного оружия) — **1×2 клетки**, чтобы лейт-билды не сжимались в 10 клеток.

## Повторы одного предмета (duplicate)

2-я, 3-я и 4-я **копия одного itemId** на доске:

- **passiveDefense** (броня): та же кривая 100% → 85% → 72% → 60%
- **урон** (damage): 2-й кинжал/посох бьёт с 85% базы
- ИИ и тултип показывают `% силы (повтор на доске)`

## Бафф оружия при блоке (grantBlockBuff)

- Cap **на каждый щит отдельно** (не общий на сторону).
- Несколько щитов с синергией: 2-й щит даёт **85%** баффа (+4 вместо +5).
- В логе: `Железный щит → +4 атаки Посох ученика (85%)`

## Классовые winrate (сим, поздние лоадауты)

Прогон: 1002 боя на класс (матрица 3×3). См. `balance-sim-results.json`.

## Изменения предметов (сводка)

**Новые utility (универсальные):** `bandage`, `lucky_charm`, `wooden_buckler`, `cork_charm`, `whetstone`, `iron_patch`, `antitoxin`, `spark_stone` — слабые, 1–4💰, тег `utility`.

**Магазин:** `rollShopBatch` — ≥1 слот по карману; ~42% шанс utility-слота; после 2 лоссов — survival-слот; неаффordable сильно занижены в весах.

**Ослаблено:** `poison_vial` (1 стак, 2💰, CD 4.5с), `poison_dagger` (1–2 урона, CD 2.6с, яд каждый 3-й удар), синергия weapon+poison (+1), `smoke_bomb` (2 яда), cap ×3, DoT ×0.5/стак, pacing 0.55.

**Усилено:** `royal_helmet`, `rune_of_magic`, `rune_of_protection`, `blood_stone`, `smoke_bomb`, `frost_crystal`, `beast_fang`, `fire_staff`, `mana_orb`, `great_shield`, `knight_sword`, `shadow_blade`, `health_stone`.

## UI / AI

- Профиль боя показывает штраф за несколько источников блока.
- Тултип предмета: `% эффективности блока (стак)`.
- ИИ снижает оценку новых block-предметов при уже имеющихся источниках.

## Магазин (экономика витрины)

| Правило | Эффект |
|---|---|
| `rollShopBatch` | Сборка всей витрины за раз с гарантиями |
| Affordable slot | Минимум 1 предмет с `cost ≤ gold` |
| Utility slot (~42%) | Один слот из пула `tags: utility` |
| После 2 лоссов | Доп. слот heal/HP/utility по карману |
| Веса | Недоступные по цене — ×0.08…0.5; utility при малом золоте — буст |
